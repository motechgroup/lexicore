LexCore Documentation

Complete guide to installing, configuring, and operating the LexCore Law Firm Portal.

Introduction

LexCore is a professional law firm management system and client intake portal designed for solo attorneys, medium-sized firms, and corporate legal departments. Built on top of the modern Laravel and Livewire frameworks, it integrates premium aesthetics, real-time client billing, interactive case folders, staff dockets, and instant notification alerts into a single cohesive platform.

Interactive Matter Dockets

Legal Billing & Invoices

Generate, itemize, and issue client statements. Complete with express online checkouts using Stripe or PayPal Express.

Notify clients and attorneys instantly regarding case filings, hearing schedules, or billing notices via Twilio SMS and SMTP Email.

Role-Based Access

System Requirements

Ensure your hosting environment meets the following specifications to run LexCore cleanly:

info
Shared Hosting Compatibility: LexCore has been optimized to execute perfectly on restrictive cPanel, Plesk, and CloudLinux environments without needing CLI or SSH server access.
  • PHP Version: PHP 8.2 or 8.3
  • Database Server: MySQL 5.7+ / MariaDB 10.3+ or SQLite 3
  • Required Extensions: OpenSSL, PDO, BCrypt/Mbstring, Tokenizer, XML, Ctype, JSON, GD (image processing), ZipArchive
  • Server Directives: Apache (with mod_rewrite enabled) or Nginx

Installation Guide

Deploying LexCore is straightforward. Choose the setup method that matches your environment:

1

Upload & Unzip

Upload the root project archives onto your public directory (e.g. public_html or your target subfolder) and extract the contents.

2

Bootstrap Environments

Verify that your document root is directed to the /public directory, or visit:
https://yourdomain.com/setup.php
Running this deployment tool creates a default .env file, establishes safe sessions/cache parameters, creates standard folder structures, clears old configs, and resets the OPCache compiledbytecode memory.

3

Run Web Installer Wizard

Navigate to https://yourdomain.com/install in your web browser. Follow the 5-step installer wizard:

  • License Verification: Enter your Envato username and valid purchase key.
  • Server Checks: Confirms required PHP modules and folder writable permissions.
  • Database Settings: Select your database driver, enter cPanel database name, database username, and password. Clicking "Test Connection" will write the uncommented variables directly to your active configuration.
  • Admin Profile: Register your super-administrator username, email, and password.

Emergency Direct Installer

In highly restrictive shared hosting domains (such as when Livewire AJAX requests are blocked by ModSecurity or if configuration cache locks prevent normal setup), you can trigger the **One-Click Standalone Installer** by calling the setup script with query parameters:

URL String Query 
https://yourdomain.com/setup.php?action=install&email=your_email@gmail.com&password=your_secure_password&name=YourName

This script runs independently from Laravel's routing system to bypass caching. It purges existing schemas, runs migrations and seeds, registers your admin profile, and locks the installer automatically.

System Modules & User Guide

Explore the three primary entry views of the LexCore system:

1. Super-Administrator Console

Provides complete operational control over the legal platform:

  • Dashboard: Aggregated analytical statistics (active matters, monthly billings, invoice collections, client distributions, upcoming dockets).
  • Cases/Matters: Full CRUD options to create lawsuit portfolios, add evidence documents, assign lead attorneys, and schedule court hearings.
  • Clients/CRM: Directory of corporate and individual clients. Edit profile credentials, search case attachments, and trace billing ledgers.
  • Staff: Register attorneys, set billing rates, assign Spatie roles/permissions, and view attorney calendars.

2. Attorney/Staff Portal

A specialized dashboard focused on case management and file updates. Attorneys can view their assigned client matters, complete assigned tasks, view upcoming court hearing dockets, and upload legal pleadings.

3. Client Portal

A secure portal for clients to access their cases:

  • Matter Files: View active litigation statuses, upcoming hearing dates, and task lists.
  • Pleadings & Files: Upload case files or photos, and download documents uploaded by their legal team.
  • Billing: View outstanding balances, download PDF invoices, and checkout securely using integrated payment gateways.

Core Configuration & Integration

Configure third-party services in your Admin Panel under System Settings:

Mail Settings (SMTP)

Required for sending automated emails. Enter your mail provider's credentials:

  • Mail Host: e.g., smtp.mailgun.org or smtp.gmail.com
  • Port: e.g., 587 (TLS) or 465 (SSL)
  • Authentication: Username and password

Payment Gateways

Enable Stripe and/or PayPal to collect retainer payments online:

  • Stripe: Enter your Stripe Publishable Key and Secret Key.
  • PayPal: Toggle sandbox or live mode, and input your client ID and client secret.

Real-Time SMS Alerts (Twilio)

Notify clients of case updates automatically: Toggle Twilio SMS gateway, and input your Twilio SID, Auth Token, and From Number.

Dynamic SEO & Analytics

Configure search engine variables dynamically on the fly:

  • SEO Title: Renders as browser tab title. Keep under 60 characters.
  • Keywords & Description: Target keywords and site snippets for Google search listings.
  • Google Analytics: Enter your Measurement ID (e.g. G-XXXXXXXXXX) to inject tracking scripts on public pages.

Troubleshooting & Support

1. Empty Username or Database in Connection Error

Cause: Laravel's configuration is cached on the server, or the .env file database credentials are commented out with leading # symbols.

Solution: Run https://yourdomain.com/setup.php. This physically deletes all files inside bootstrap/cache/ to clear cached data, and resets memory caches.

2. Writable Permissions Issues

Cause: Storage directories do not have write access.

Solution: Log in to your cPanel File Manager and set permissions for storage/ and bootstrap/cache/ to 775 or 755.

3. Livewire AJAX Requests Blocked

Cause: Some shared hosting Web Application Firewalls (such as ModSecurity) block Livewire POST requests.

Solution: Ask your host to whitelist Livewire routes, or add the following directive to your root .htaccess file to disable ModSecurity:

.htaccess Directive 
<IfModule mod_security.c>
    SecFilterEngine Off
    SecFilterScanPOST Off
</IfModule>
